# 1. 共通の起動テンプレート

これは最初に運用を始めるとき、または新しい案件で最初から始めるときのテンプレート。

```markdown
あなたは、シナリオテスト設計運用のAIエージェントです。

以下のファイルを参照して動作してください。
- rules.md
- orchestrator.md
- step1_input_summary_guide.md
- step2_viewpoint_identification_guide.md
- step3_business_flow_guide.md
- step4_scenario_discovery_guide.md
- step5_testcase_detail_guide.md
- step6_priority_guide.md
- step7_csv_export_guide.md

今回の対象資料は以下です。
- [資料名1]
- [資料名2]
- [資料名3]

補足:
- [対象システムや業務の簡単な説明]
- [今回の対象範囲]
- [対象外があれば記載]

動作ルール:
- rules.md を最優先で守ること
- orchestrator.md に従って現在ステップを判断すること
- 成果物を出力したら必ず停止すること
- 次のステップへ勝手に進まないこと
- 不明点は要確認事項として明示すること

まず Step1 を実施してください。
```

# 2. レビュー・修正依頼テンプレート

各ステップで、出てきた成果物を直させるためのテンプレート。
これがあると運用がかなり安定する。

## 軽微修正テンプレート

```markdown
前回出力した成果物について、以下を修正してください。

対象ステップ:
- [Step番号]

修正内容:
- [修正点1]
- [修正点2]
- [修正点3]

修正ルール:
- それ以外の部分は原則維持すること
- 既存IDは可能な限り維持すること
- 影響範囲がある場合は要確認事項に明記すること
- 修正版を出力したら停止すること
```

---

## 観点追加テンプレート

```markdown
Step2 のテスト観点一覧について、以下の観点が不足しているため追加・見直ししてください。

追加または見直したい論点:
- [論点1]
- [論点2]
- [論点3]

要件:
- 既存観点との重複を整理すること
- 必要なら統合や分割を行うこと
- 根拠資料も明記すること
- 修正版を出力したら停止すること
```

---

## 業務フロー見直しテンプレート

```markdown
Step3 の業務フロー一覧について、以下の点を見直してください。

見直し内容:
- 業務順序の修正: [内容]
- アクター定義の修正: [内容]
- 分岐条件の追加・修正: [内容]
- 関連 viewpoint_id の修正: [内容]

要件:
- 既存IDは可能な限り維持すること
- 影響が Step4 以降に及ぶ場合は明記すること
- 修正版を出力したら停止すること
```

---

## シナリオ粒度調整テンプレート

```markdown
Step4 のシナリオ候補一覧について、粒度を見直してください。

見直し方針:
- 統合したいシナリオ: [SCN-xxx, SCN-yyy]
- 分割したいシナリオ: [SCN-zzz]
- 追加したい異常系: [内容]

要件:
- 正常系・異常系のバランスを保つこと
- related_business_flow_ids と related_viewpoint_ids を維持または見直すこと
- 修正版を出力したら停止すること
```

---

## expected_results 強化テンプレート

```markdown
Step5 のテストケース一覧について、expected_results が曖昧なので見直してください。

見直し対象:
- [SCN-001]
- [SCN-004]
- [SCN-010]

見直し方針:
- 「正しく表示される」「正常に完了する」などの曖昧表現を避ける
- 状態変化、データ反映、画面反映、メッセージ、連携結果など、確認可能な内容へ具体化する
- steps との対応が分かるようにする

修正版を出力したら停止してください。
```

---

## 優先順位見直しテンプレート

```markdown
Step6 の優先順位付けについて、以下の観点で見直してください。

見直し内容:
- Crit が多すぎるため相対評価を見直す
- 以下のケースは High ではなく Mid 相当ではないか確認する:
  - [SCN-xxx]
  - [SCN-yyy]
- 以下のケースは Low ではなく High 相当ではないか確認する:
  - [SCN-zzz]

要件:
- priority_rationale もあわせて見直すこと
- テストケース本文は原則変更しないこと
- 修正版を出力したら停止すること
```

# 3. 一気通し運用を避けるための短い指示文

毎回長々書きたくない場合の、かなり短い差し込み文も作っておくと便利。

## 次へ進む指示

```markdown
前回成果物を承認します。次のステップへ進んでください。
```

## 同一ステップの再修正指示

```markdown
前回成果物は未承認です。以下を修正して同じステップを再出力してください。
- [修正点]
```

## ここだけ重点的に見直す指示

```markdown
全体は維持しつつ、以下だけ重点的に見直してください。
- [対象]
- [修正方針]
```

# 6. Roo Code などで使いやすい最小テンプレート

## Step実行の最小形

```markdown
rules.md, orchestrator.md, stepX_guide.md を参照してください。
前工程成果物:
- [成果物名]

入力資料:
- [資料名]

今回の依頼:
[Step名] を実施してください。

要件:
- 該当step guideに従うこと
- 出力後は必ず停止すること
- 不明点は要確認事項として明示すること
```

## 修正の最小形

```markdown
前回の [Step名] 成果物を修正してください。

修正点:
- [修正点1]
- [修正点2]

要件:
- 既存IDは可能な限り維持
- 影響範囲があれば明記
- 修正版を出力したら停止
```
