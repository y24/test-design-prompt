# run_step_prompt_template.md

## 1. 目的

本ファイルは、非機能に関する品質基準策定プロセスを **1ステップずつ実行するための共通プロンプトテンプレート** である。  
AI エージェントは、本テンプレートを起点として、指定されたステップだけを実行し、成果物を出力した時点で停止する。

本テンプレートの目的は、以下である。

- `rules.md` / `orchestrator.md` / 各 `step guide` の読込を明示する
- 実行対象ステップを明確に指定する
- 入力資料と既存成果物を明確に渡す
- issue log 更新を毎回忘れないようにする
- 1ステップ実行・停止の運用を崩さないようにする

---

## 2. 使い方

このテンプレートは、各ステップ開始時に使用する。  
以下のような場面を想定する。

- Step 1 を最初に開始するとき
- Step 2 以降を、前ステップのレビュー完了後に開始するとき
- 途中で資料追加後、特定ステップを再実行するとき

本テンプレートは **1回につき1ステップ分 בלבד** 使用する。  
複数ステップをまとめて進めるためには使わない。

---

## 3. 実行前提

このテンプレートを使う AI エージェントは、以下を守ること。

- `rules.md` を共通原則として参照する
- `orchestrator.md` を進行制御として参照する
- 指定された `step guide` のみを今回の主対象として実行する
- 不足情報は推測で補完しない
- 必要な issue は `issue_log_template.md` に従って整理する
- 成果物出力後は必ず停止する
- ユーザーの明示的な指示があるまで次ステップへ進まない

---

## 4. 共通テンプレート

以下を、実行時のプロンプトとして用いる。

```text
あなたは、非機能に関する品質基準（品質クライテリア）を段階的に策定する AI エージェントである。

以下のファイルを参照して実行すること。

【共通ルール】
- rules.md
- orchestrator.md

【補助ファイル】
- issue_log_template.md
- output_schema_examples.md
- 必要に応じて quality_characteristic_scope_notes.md
- 必要に応じて evidence_reference_style.md

【今回実行する step guide】
- {{STEP_GUIDE_FILE}}

【入力資料】
{{INPUT_DOCUMENTS}}

【既存成果物】
{{EXISTING_OUTPUTS}}

【既存 issue log】
{{EXISTING_ISSUE_LOG}}

【ユーザーからの今回の指示】
{{USER_INSTRUCTION}}

【今回の実行目的】
{{EXECUTION_PURPOSE}}

実行にあたっては、以下を必ず守ること。

1. 今回は {{STEP_NAME}} のみを実行すること
2. rules.md の共通原則に従うこと
3. orchestrator.md の進行制御に従うこと
4. 指定 step guide の目的・手順・出力方針に従うこと
5. 資料不足や矛盾がある場合、推測で補完しないこと
6. 必要な issue を抽出し、issue_log_template.md に従って整理すること
7. 出力では、根拠資料・不足情報・関連 issue を可能な限り明示すること
8. 成果物を出力したら必ず停止すること
9. ユーザーの明示的な次指示があるまで、次ステップへ進まないこと

出力は、少なくとも以下の順で構成すること。

1. 現在ステップ名
2. 当該ステップの成果物
3. issue 追加・更新内容
4. レビュー観点
5. 停止

なお、今回は次ステップの内容を出力しないこと。
````

---

## 5. プレースホルダ説明

テンプレート内の各プレースホルダは、以下の意味で使用する。

| プレースホルダ                  | 内容                     |
| ------------------------ | ---------------------- |
| `{{STEP_NAME}}`          | 実行対象ステップ名              |
| `{{STEP_GUIDE_FILE}}`    | 実行対象の step guide ファイル名 |
| `{{INPUT_DOCUMENTS}}`    | 今回参照させる入力資料一覧          |
| `{{EXISTING_OUTPUTS}}`   | 直前までの成果物               |
| `{{EXISTING_ISSUE_LOG}}` | 現在の issue log          |
| `{{USER_INSTRUCTION}}`   | ユーザーからの今回の指示           |
| `{{EXECUTION_PURPOSE}}`  | 今回そのステップで何を達成したいか      |

## 9. 運用上の注意

### 9.1 実行対象ステップを必ず明示する

「続けて」だけだと、AI が妙に張り切って複数ステップ進めたくなることがある。
機械はときどき親切の皮をかぶった暴走列車になるので、ステップ名は明示したほうが安全。

### 9.2 既存成果物は要約で渡してよい

全文再貼り付けが重い場合は、既存成果物を要約で渡してよい。
ただし、ID や未確定事項、issue 番号は落とさないこと。

### 9.3 issue log は差分だけでもよい

issue log が長くなった場合は、全件ではなく以下を優先して渡してよい。

* open
* pending-review
* accepted-undetermined
* 今回のステップに影響する resolved

### 9.4 入力資料は版数や改訂日を入れる

同名資料の版違いがあると、地味に事故る。
版数や改訂日が分かるなら入れておくほうがいい。

### 9.5 次ステップを示唆しない

テンプレート内で「終わったら次は Step X へ」などとは書かないこと。
進行制御は `orchestrator.md` に任せ、今回の実行は今回のステップで閉じる。

---

## 10. 本ファイルの責務外

本ファイルは、**ステップ開始時の共通呼び出しテンプレート** を示すものであり、以下は扱わない。

* レビュー後に同一ステップを再実行するための差分反映テンプレート
* レビュー後に次ステップへ進むための再開テンプレート
* issue log だけを更新するための専用テンプレート
* 実際の品質基準策定成果物そのもの

それらは別テンプレートとして分ける。
