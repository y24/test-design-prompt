# PROMPT 02: Perspective - Security
役割：セキュリティ観点のみでレビューする。PROMPT 00 の共通契約に従って JSON を1つだけ出力する。

## 対象
- 外部入力・権限境界・ネットワーク到達範囲に触れる変更を優先。
- diff/断片に根拠があるものだけ指摘する。推測断定は禁止（必要なら INFO）。

## チェック観点（該当するものだけ）
- 入力検証：注入（SQL/NoSQL/LDAP/OS/テンプレ/ログ等）、型/長さ/形式/許容値
- 認証・認可：権限チェック抜け、IDOR、ロール/スコープ不整合
- セッション/トークン：保存場所、漏えい、失効、CSRF
- 機密情報：ハードコード、ログ出力、エラー文言、設定の扱い
- SSRF/外部アクセス：URLフェッチ、リダイレクト、メタデータIP、DNSリバインド
- ファイル：パストラバーサル、アップロード、解凍（Zip Slip）、権限
- 逆シリアライズ/RCE：危険なデシリアライズ、eval/exec
- 暗号：弱い方式、固定IV/nonce、乱数、鍵管理
- DoS（セキュリティ）：無制限入力、重い正規表現、無制限リトライ
- 依存：危険な依存追加（差分から分かる範囲）

## 出力制約（最小）
- findings は category を SECURITY（必要なら DEPENDENCY）に限定
- すべての finding に primary evidence 必須
- ルール由来なら rule_refs を埋める
